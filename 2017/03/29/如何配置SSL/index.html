什么是SSL？

SSL(Secure Sockets Layer 安全套接层),及其继任者传输层安全（Transport Layer Security，TLS）是为网络通信提供安全及数据完整性的一种安全协议。TLS与SSL在传输层对网络连接进行加密。

为什么使用SSL?

机密性：SSL协议使用密钥加密通信数据。

可靠性：服务器和客户都会被认证，客户的认证是可选的。

完整性：SSL协议会对传送的数据进行完整性检查。

在Startssl 上申请 SSL证书

  • 首先登陆到startssl并注册，两种登陆方式：证书登陆，需要你安装证书，还有一种就是发送密码到你的注册邮箱

  • 登陆成功后会跳到如下页面，首先验证域名证明域名归你所有,点击

    Validations Wizard填写域名并提交

  • 进入域名所有者信息验证页，选择你的邮箱会发送一个验证码到你的邮箱，填写验证码提交，注册成功

    可能有的人找不到自己的邮箱，这是你开启了域名隐私保护，去域名管理的地方关闭一下，重新提交就找到你的邮箱了

  • 点击Certificates Wizard,填写你的域名，子域名等，最多可以添加5个

  • 填写域名后，下面需要生成CSR文件并且填入提交

  • 提交成功后在返回的页面点击选中地方，下载一个文件

  • 文件是各种服务器需要的crt证书，我选择的是Nginx证书，解压后的文件用SCP发送到服务器

在Nginx配置SSL 证书

  • 配置SSL证书需要两个文件一个是解压赌赢服务器文件所得的证书，例如：选择的NginxServer.zip解压缩的文件

  • 另一个文件是解密我们生成的key文件所得的内容

    点击Tool Box 选择Decrypt Private Key 复制cat出的数据到 Private key,

    Passphrase 输入之前生成文件时候设置的PEM pass phrase，点击Decrypt 复制解密的内容 粘贴到后缀名为.key的文件

  • 配置nginx

SSL等级检测

访问https://www.ssllabs.com/ssltest/ ,填写域名点击提交大概两分钟左右，输出如下信息

等级是B，有的是C，可以看看对应等级所欠缺的

提高SSL等级

  • GitHub 上有一个开源项目server-configs-nginx 地址是我fork的项目，server-configs-nginx/h5bp/directive-only/ssl.conf 的文件 复制这三行到你的配置内；在测试应该可以达到A

  • Github上一个开源的项目mozilla/server-side-tls 会基于不同服务器版本生成不同增强型 SSL 配置；并且加入 HSTS 特性支持.

    HSTS的作用是强制客户端（如浏览器）使用HTTPS与服务器创建连接。
